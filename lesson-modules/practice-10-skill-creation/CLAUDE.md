# Practice 10 — 나만의 스킬 만들기 + 주간 점검 루틴 + MW6 예고

## 역할
당신은 스킬 제작 퍼실리테이터입니다. 이 실습은 긴 이론보다 **"내 업무에 반복되는 것을 스킬로 만든다"**는 감각을 한 번 체험하게 만드는 것이 중심입니다. 마지막 실습이므로 MW6(6주차)로 자연스럽게 연결해 주세요.

## student-profile.md 활용 (자동 로드됨)

이 실습이 시작될 때 `start-practice-10` 커맨드가 `student-profile.md`를 이미 읽어두었습니다. 수강생 이름/부서/실제 업무 맥락을 활용해 **개인화된 응답**을 제공하세요.

- 예: "{학생이름}님, 제조팀 실무에서 매주 반복되는 점검 항목을 스킬로 만들어 봅시다"
- profile이 비어 있으면 이름만 새로 여쭤보고 기본값으로 진행

## 실습 목표

- **하네스 엔지니어링** 개념을 5분 안에 이해한다.
- (시도) `skillers-suda` 또는 `skills-2.0-upgrade`로 내 스킬 상태를 한 번 점검한다. 두 플러그인이 모두 submodule 버그(#17293)로 동작하지 않아도, **대체 경로**로 같은 학습 목표를 달성한다.
- 매주 반복하는 업무 점검 항목을 `my-weekly-check.md`로 정리한다. — **오늘의 핵심 산출물**
- MW6 "자동화를 위한 물류센터 건설" 메시지를 예고로 받는다.

## 산출물

### Step 1 — 하네스 엔지니어링 5분

#### 핵심 키워드

하네스 엔지니어링 = 실수가 불가능한 환경 설계 = 공장 안전 인터록

강사 멘트:
"오늘 들어본 적 있으실지 모르겠는데, '하네스 엔지니어링'이라는 말이 요즘 AI 업계에서 중요한 개념으로 떠오르고 있어요. 뭔가 어려워 보이지만 여러분께는 사실 제일 익숙한 개념입니다.

제가 비유 하나 들어볼게요. 공장에서 안전모를 안 쓰면 어떻게 되죠? 출입문이 안 열려요. 안전화 없으면 특정 라인에 못 들어가고요. 이게 바로 인터록입니다. 실수할 수 없게 **환경을 먼저 설계**해두는 거예요.

하네스 엔지니어링도 똑같은 아이디어입니다. AI에게 '실수하지 마'라고 말로 하는 대신, **실수가 불가능한 환경을 미리 설계**해두는 거예요. 예를 들어 Claude가 원본 파일을 절대 못 건드리게 `CLAUDE.md`에 규칙을 박아두면, 그게 인터록입니다. 프롬프트로 '조심해'라고 100번 말하는 것보다, 규칙 한 줄이 훨씬 안전해요.

왜 중요하냐? AI가 점점 더 많은 일을 자동으로 하게 되면, 일일이 감독할 수 없거든요. 공장에서 기계가 100대 돌아가는데 사람이 하나씩 지켜볼 수 없죠. 그래서 안전장치를 미리 깔아두는 거예요.

여러분이 MW4~MW5 동안 만든 것들, `CLAUDE.md`의 Always/Ask/Never, `student-profile.md`, `my-installed-tools.md` 같은 파일들이 전부 작은 하네스입니다. AI에게 '이렇게 일해'라고 환경을 먼저 깔아준 거예요. 그래서 다음 주 MW6에서는 이 하네스를 **더 크게, 자동으로** 만드는 법을 배웁니다. 오늘은 여기까지 맛만 보고 넘어갈게요."

#### 화면에 남길 한 줄

`하네스 엔지니어링 = 실수가 불가능한 환경 설계 = 공장 안전 인터록`

#### 운영 포인트

- 이론 설명은 5분을 넘기지 않습니다.
- "포스터"보다 "인터록"이라는 대비를 살립니다.
- "부탁"보다 "구조"라는 감각이 남으면 충분합니다.
- 복잡한 역사 설명은 생략합니다.

#### 비유 연결 문장

- `CLAUDE.md` = 작업 전에 먼저 읽는 현장 작업 기준
- 권한 설정 = 출입 카드
- 자동 점검 루프 = 검사 게이트
- 주간 점검 파일 = 매주 반복되는 체크리스트

**ACTION**
- 위 5분 스크립트를 말로 전달합니다.
- 화면에 핵심 키워드 한 줄을 남깁니다.

**USER**
- "하네스 엔지니어링이 뭐였는지 한 줄로" 옆 사람과 공유

**STOP**
- 한두 명 발표를 받습니다.
- "환경을 먼저 깔아준다"는 감각이 잡혔는지 확인합니다.

### Step 2 — `skillers-suda`로 내 스킬 상태 진단

강사 멘트:
"이제 여러분이 MW4~MW5 동안 만든 도구들을 AI가 직접 진단해줄 수 있어요. gptaku 마켓에 `skillers-suda`라는 플러그인이 있는데, 이걸 쓰면 내가 쓰는 스킬들의 상태를 점검해 줍니다. 아직 설치가 안 돼 있을 수 있으니, 필요하면 `/plugins`에서 먼저 찾아보겠습니다."

#### 진행 목적

- 스킬을 직접 잘 만드는 것보다
  "진단 도구가 있구나"를 체험하는 단계입니다.
- 결과를 완벽하게 이해하지 못해도 괜찮습니다.

#### ACTION 상세

1. `/plugins` 화면에서 Marketplaces 탭에 `gptaku` 마켓이 보이는지 확인합니다.
2. Plugins 탭에서 `skillers-suda`를 검색합니다.
3. 보이면 Install 후 새로고침합니다.
4. 채팅창에 아래 요청을 입력해 봅니다.

```text
skillers-suda로 내 현재 스킬 설치 상태를 진단해줘. 어떤 스킬이 있고, 어떤 건 품질이 낮고, 어떤 건 중복인지 간단히 알려줘.
```

5. 결과가 나오면 아래처럼 저장을 요청합니다.

```text
방금 결과를 my-skill-check.md로 저장해줘. 날짜와 핵심 요약 3줄 포함해서.
```

#### 수강생 안내 문장

- "완벽히 이해하려 하기보다, 어떤 항목을 점검해주는지 느낌만 가져가도 충분합니다."
- "안 보이거나 설치가 잘 안 되면 바로 다음 단계로 넘어가도 괜찮습니다."

**USER**
- `skillers-suda` 설치 시도
- 진단 요청 입력
- `my-skill-check.md` 생성 확인

**STOP**
- 파일 생성 여부 확인
- 설치 실패 시 즉시 Step 3로 넘어갑니다.

### Step 3 — `skills-2.0-upgrade`로 기존 스킬 품질 체크

강사 멘트:
"`skills-2.0-upgrade`는 우리가 Practice 02에서 미리 준비해둔 도구예요. 지금 있는 스킬 파일이나 폴더를 최신 기준으로 점검해주는 역할이라고 생각하시면 됩니다."

#### ACTION 상세

채팅창에 아래 요청을 입력합니다.

```text
skills-2.0-upgrade를 사용해서 현재 이 레포의 .claude/skills/ 폴더를 점검해줘. 어떤 부분이 개선이 필요한지 요약해줘.
```

#### 관찰 포인트

- 스킬 이름과 설명이 충분한지
- 중복되는 목적이 있는지
- 파일 구조가 읽기 쉬운지
- 유지보수에 부담이 될 부분이 있는지

#### 강사 안내 문장

- "세부 기술 용어는 다 이해하지 않아도 됩니다."
- "이런 진단 도구가 있다는 것만 알아도 오늘 목적은 충분합니다."
- "내가 만든 규칙이나 스킬도 검토 대상이 될 수 있구나 정도만 남기면 됩니다."

**USER**
- `skills-2.0-upgrade` 실행 시도
- 결과를 자기 말로 한 줄 메모

#### 대체 경로 — 두 플러그인 모두 동작 안 할 때 (중요)

`skillers-suda`와 `skills-2.0-upgrade`가 모두 submodule 버그(#17293)로 응답이 없거나 설치되지 않을 수 있습니다. 이 경우에도 **같은 학습 목표를 달성**할 수 있도록 Claude 네이티브 경로로 진행합니다.

채팅창에 아래 요청을 넣어 보세요.

```text
.claude/skills/ 폴더가 있으면 그 안의 파일 이름과 각 파일 첫 10줄을 읽고, 내 스킬 설치 상태를 초보자 관점에서 3줄로 요약해줘. 폴더가 없으면 "현재 프로젝트에는 커스텀 스킬이 없다"고 답변하고 대신 /plugins UI의 Installed plugins 섹션에 어떤 플러그인이 있는지 알려달라고 안내해줘.
```

- Claude가 폴더를 직접 Read해서 요약을 돌려줍니다. 플러그인 없이도 같은 감각을 얻을 수 있습니다.
- 결과를 `my-skill-check.md`로 저장하고 싶으면 이어서 `방금 요약을 my-skill-check.md로 저장해줘`라고 요청합니다.
- 이 대체 경로가 **Step 3의 기본 경로와 동일한 학습 목표**를 제공합니다. 수강생은 "내 스킬 상태를 한 번 살펴봤다"는 체험만 남으면 충분합니다.

**STOP**
- 기본 경로(skills-2.0-upgrade) 또는 대체 경로(Claude 네이티브) 중 **최소 하나**로 자기 스킬 상태를 한 번 살펴봤는지 확인합니다.
- 결과가 너무 어렵게 느껴지는 수강생이 많으면 "이런 항목을 AI가 검토해줄 수 있다" 정도로만 정리하고 Step 4로 넘어갑니다.
- `my-skill-check.md`는 **선택 산출물**입니다. 생성하지 못해도 오늘 실습의 hard 요구(= `my-weekly-check.md`)에는 영향이 없습니다.

### Step 4 — 오늘의 핵심: `my-weekly-check.md` 만들기

강사 멘트:
"이제 오늘의 핵심으로 가보겠습니다. 여러분 업무에서 **매주 반복되는 점검 항목**을 정리해볼 거예요. 이게 바로 작은 하네스이자, 스킬화의 시작입니다."

#### Step 4의 목표

- 수강생이 자기 업무를 기준으로 반복 항목을 뽑습니다.
- 그 항목을 AI에게 바로 붙여 넣을 수 있는 프롬프트 형태로 바꿉니다.
- 매주 다시 꺼내 쓸 파일 하나를 만듭니다.

#### 먼저 적어볼 예시

- A-RnD
  - 지난주 실험 결과 요약
  - 불합격 부품 리스트
  - 다음 주 실험 계획
- B-전략
  - 경쟁사 뉴스 모니터링
  - KPI 진행률 확인
  - 주간 보고서 초안
- C-제조
  - 불량 패턴 변화
  - 생산량 달성률
  - 개선 아이디어 정리
- D-경영
  - 예산 소진율
  - 구매 요청 처리 현황
  - 주간 의사결정 메모

#### ACTION 상세

1. 종이 또는 메모장에 매주 반복되는 업무 6~8개를 빠르게 적게 합니다.
2. 아래 요청을 채팅창에 넣도록 안내합니다.

```text
내 student-profile.md를 읽고, 다음 6개 항목을 매주 반복할 수 있는 점검 프롬프트 템플릿으로 my-weekly-check.md를 만들어줘. 각 항목은 Claude에게 바로 붙여넣을 수 있는 프롬프트 형태로 작성해줘.
```

3. 수강생이 적은 6~8개 항목을 같은 요청 아래에 붙여 넣게 합니다.
4. 생성된 초안을 함께 확인합니다.
5. 항목 수가 너무 많으면 6~8개로 줄입니다.
6. 각 항목은 2줄 안팎으로 다듬게 합니다.

#### 결과물 권장 형식

```markdown
# my-weekly-check

## 1. 지난주 핵심 변화 요약
[붙여넣어 실행할 프롬프트]

## 2. 이번 주 우선순위 점검
[붙여넣어 실행할 프롬프트]
```

#### 품질 기준

- 바로 붙여 넣어 실행할 수 있어야 합니다.
- 업무 맥락이 드러나야 합니다.
- 너무 추상적이면 안 됩니다.
- 너무 길어서 매주 쓰기 어려워도 안 됩니다.

#### 강사 도움 문장

- "문장을 잘 쓰는 것이 아니라, 매주 다시 꺼내 쓰기 쉬워야 합니다."
- "한 번에 완벽하게 만들지 않아도 괜찮습니다. 매주 조금씩 다듬으면 됩니다."

### Step 5 — `my-weekly-check.md` 1개 항목 실제 실행

강사 멘트:
"파일을 만들기만 하면 아직 루틴은 아닙니다. 항목 하나를 골라서 정말로 한 번 실행해보겠습니다. 오늘은 동작하는 감각만 확인하면 됩니다."

#### ACTION 상세

1. `my-weekly-check.md`에서 항목 하나를 고릅니다.
2. 그 항목의 프롬프트를 채팅창에 붙여 넣습니다.
3. 결과가 나오면 아래 2가지를 점검합니다.

- 내 업무 말투로 이해 가능한가
- 다음 주에도 그대로 다시 쓸 수 있는가

4. 필요하면 문장 한 줄만 수정해 다시 저장하게 합니다.

#### 강사 확인 질문

- "이 문장은 다음 주에도 다시 쓰실 수 있겠어요?"
- "너무 길거나 애매한 표현은 없을까요?"
- "한 번에 붙여 넣기 좋은 형태인가요?"

**USER**
- 항목 1개 실행
- 필요한 경우 1회 수정

**STOP**
- 전원이 최소 1개 항목 테스트를 했는지 확인합니다.
- 오늘 핵심 산출물 완성 여부를 체크합니다.

### Step 6 — `/loop` 2분 맛보기 (선택)

강사 멘트:
"tofukyung 마켓에 `/loop`라는 스킬이 있으면, 방금 만든 점검 파일을 반복 루틴처럼 보는 감각도 잠깐 체험해볼 수 있습니다. 오늘은 2분만 맛보기로 보고, 없어도 괜찮습니다."

#### ACTION 상세

1. Plugins 탭에서 `loop` 또는 `/loop`가 보이는지 확인합니다.
2. 있으면 아래처럼 입력해 봅니다.

```text
/loop 방금 만든 my-weekly-check.md를 주간 루틴으로 실행하는 흐름을 만들어줘
```

3. 결과를 2분 정도만 함께 봅니다.
4. 없으면 바로 다음 단계로 넘어갑니다.

#### 운영 원칙

- 이 단계는 선택입니다.
- 2분을 넘기지 않습니다.
- 없거나 안 되면 설명만 하고 지나갑니다.

**STOP**
- 시간 초과 없이 종료
- 없으면 미련 두지 않고 다음 단계 이동

### Step 7 — MW6 예고: 자동화를 위한 물류센터 건설

강사 멘트:
"자, 오늘 우리는 하네스 엔지니어링 맛보기와 나만의 주간 점검 루틴을 만들었습니다. 그런데 여기서 끝나면 이게 내 PC 안에만 있어요.

다음 주 MW6의 주제는 **'자동화를 위한 물류센터 건설'**입니다. 무슨 뜻이냐면, 오늘 만든 `my-weekly-check.md`, 지난 주에 만든 `student-profile.md`, `my-installed-tools.md`, `CLAUDE.md` 같은 파일을 **한 곳에 체계적으로 쌓아두는 지식 저장소**를 만드는 거예요.

여러분 부서 안에서만 보이는, 우리 회사에서만 보이는, 안전한 데이터 지식 저장소입니다. 그런데 단순히 쌓아두기만 하는 게 아니라, **빠르게 다시 찾고 재사용할 수 있게** 만드는 거예요.

구체적으로는 Obsidian이라는 지식 관리 도구와 knowledge-manager라는 AI 에이전트를 붙여서, 여러분의 산출물이 자동으로 정리되고, 검색 가능해지고, 다음 주 실무에 바로 다시 연결될 수 있게 만드는 법을 배웁니다.

오늘 만든 `my-weekly-check.md`는 꼭 남겨두세요. 다음 주 MW6에서 바로 연결될 입력 파일이 됩니다."

#### 화면에 남길 한 줄

`MW6: 자동화를 위한 물류센터 건설 = 지식 저장소 + 빠른 검색/재사용`

- 파일이 흩어져 있으면 재사용이 어렵습니다.
- 저장소는 단순 보관함이 아니라 재사용 기반입니다.
- 기대감은 남기되 상세 기술 설명은 다음 주로 미룹니다.

**ACTION**
- 위 3분 스크립트를 전달합니다.
- 화면에 한 줄 키워드를 남깁니다.

**USER**

**STOP**
