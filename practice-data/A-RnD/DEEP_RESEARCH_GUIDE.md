# A-RnD 그룹 Deep Research 가이드

> 3교시 실무 반복 루프에서 2차 데이터만으로 부족할 때 사용하는 외부 리서치 가이드입니다.
> 이 문서는 VSCode extension 채팅창의 `/deep-research` 슬래시 커맨드 사용을 전제로 합니다.
> 내부 로그와 해석 결과를 먼저 보고, 외부 정보는 보조 근거로 붙이는 흐름을 권장합니다.

## 언제 쓰나요?

- 3교시 "부서별 실무 반복 루프 5단계" 중 "데이터" 단계에서 `practice-data/A-RnD/` 내부 파일만으로 맥락이 부족할 때
- 최신 배터리 논문, 경쟁사 CAE 도구, 전고체 배터리 동향 등 외부 정보가 필요할 때
- 강의장 네트워크에서 VSCode extension의 `/deep-research` 기능이 가용한 경우
- `battery_log.csv` 또는 `battery_log_2026Q2.csv`에서 이상 이벤트가 보였는데 원인 가설이 여러 개인 경우
- `result.csv`의 안전계수, 소재 선택, 두께 판단을 최근 기술 흐름과 연결해 보고 싶을 때

## 먼저 확인할 내부 파일

- `practice-data/A-RnD/battery_log.csv`
- `practice-data/A-RnD/battery_log_2026Q2.csv`
- `practice-data/A-RnD/result.csv`

## VSCode extension에서 /deep-research 호출하기

superpowers 플러그인을 설치했다면 VSCode 채팅창에 아래처럼 요청합니다.

```text
/deep-research 전고체 배터리 2026 Q1 최신 연구 동향 요약해줘
```

또는 이렇게 조금 더 명시적으로 적을 수 있습니다.

```text
superpowers:deep-research 스킬로 배터리 열폭주 방지 설계 사례를 조사해줘
```

목적까지 같이 적으면 답변 품질이 더 안정적입니다.

```text
/deep-research 배터리 셀 저전압 이상 탐지 기준을 최근 논문 기준으로 정리해줘. 제조업 실무자가 바로 비교할 수 있게 표로 요약해줘
```

## A-RnD 권장 쿼리 예시

1. `/deep-research 2026 전고체 배터리 양산 기술 최신 논문 3개 요약`
2. `/deep-research 리튬이온 배터리 열폭주 방지 CAE 시뮬레이션 최신 사례`
3. `/deep-research 자동차용 배터리 팩 구조 설계 2026 트렌드`
4. `/deep-research 배터리 안전계수 판정 기준 업계 표준 최근 변화`
5. `/deep-research 전기차 배터리 수명 예측 머신러닝 모델 최근 논문`
6. `/deep-research 배터리 셀 전압 3.0V 미만 경고 기준을 최근 1년 자료로 설명해줘`
7. `/deep-research 45도 이상 셀 온도 상승 시 열화 리스크를 설명한 최신 연구를 찾아줘`

## 추천 입력 템플릿

```text
/deep-research {주제}를 조사해줘.
대상은 자동차 제조업 R&D 실무자이고,
최근성은 최근 12개월 우선,
출처 URL과 발표일을 함께 적어주고,
핵심 주장 3개와 실무 적용 포인트 3개를 분리해서 정리해줘.
```

## 결과를 기존 분석에 합치는 법

1. `/deep-research` 결과가 나오면 `외부리서치_{날짜}.md`로 먼저 저장합니다.
2. `battery_log.csv` 또는 `battery_log_2026Q2.csv` 분석 요청 시 함께 맥락으로 전달합니다.
3. 예: "`battery_log_2026Q2.csv` 이상 이벤트를 `외부리서치_2026-04-06.md`의 최신 논문 기준과 비교해서 분석해줘"
4. `result.csv`를 함께 볼 때는 구조 안전성, 소재, 두께, 안전계수까지 연결해 묻습니다.

## 추천 작업 순서

1. 내부 데이터에서 이상 징후를 먼저 찾습니다.
2. 이상 징후를 한 줄 질문으로 바꿉니다.
3. VSCode 채팅창에서 `/deep-research`를 실행합니다.
4. 결과에서 출처와 날짜를 먼저 확인합니다.
5. 내부 데이터와 직접 연결되는 문장만 골라 메모합니다.
6. 발표나 보고용 문장으로 짧게 다시 씁니다.

## battery_log 분석과 연결하는 질문 예시

- `CELL-A03`의 2.88V는 일반적인 저전압 경고 기준과 비교하면 어느 수준인가요?
- 45도 이상 온도 상승이 반복되면 수명과 안전성에 어떤 영향을 줄 수 있나요?
- 동일 셀에서 전압 저하와 고온이 함께 나타날 때 자주 언급되는 원인은 무엇인가요?
- 열폭주 조기 징후를 논문에서는 어떤 센서 조합으로 다루나요?

## result.csv 분석과 연결하는 질문 예시

- 배터리 케이스 상판과 하판에서 최근 경량화 소재 선택은 어떤 방향인가요?
- 안전계수 2.0 전후가 실무적으로 어떤 의미를 가지는지 설명해줄 수 있나요?
- 충돌 하중과 열관리 설계를 함께 다룬 사례가 있나요?
- Al5052와 Al6061 적용 차이를 설명한 최신 자료가 있나요?

## 좋은 질문과 아쉬운 질문

좋은 질문 예시:

- `/deep-research 배터리 셀 저전압 이상 탐지 기준을 최근 1년 논문 기준으로 3개만 요약해줘`
- `/deep-research 자동차용 배터리 팩 열폭주 확산 방지 설계의 최신 검증 방법을 표로 정리해줘`
- `/deep-research 배터리 하우징 경량화 소재 비교를 제조업 실무자 관점으로 설명해줘`

아쉬운 질문 예시:

- `/deep-research 배터리 알려줘`
- `/deep-research 최신 기술 전부 정리`
- `/deep-research 뭐가 중요해?`

## 출처 기록 방식

```markdown
## 참고 출처
- 제목:
- 발행기관:
- 발표일:
- URL:
- 핵심 한 줄:
```

## 실습에 맞는 정리 포맷

```markdown
## 외부 리서치 요약
- 조사 주제:
- 확인 날짜:
- 핵심 발견 1:
- 핵심 발견 2:
- 우리 데이터와 연결 포인트:
```

## 검증 체크리스트

- 결과에 날짜가 적혀 있는가
- 출처 URL이 포함되어 있는가
- 논문, 기업 발표, 기관 보고서가 구분되어 있는가
- 내부 데이터와 직접 연결되는 문장이 있는가
- 숫자 단위가 명확한가
- 내가 다시 설명할 수 있을 만큼 이해했는가

## 자주 쓰는 주제 묶음

- 셀 이상 탐지 기준
- 고온 이벤트 해석
- 팩 구조 경량화
- 배터리 하우징 소재 선택
- CAE 자동화 동향
- 전고체 배터리 양산 이슈

## 실무 연결 문장 예시

- "외부 논문에서는 3.0V 미만을 저전압 경고 수준으로 다루는 사례가 많아, Q2 로그의 저전압 이벤트는 우선 점검 대상으로 볼 수 있습니다."
- "최근 열관리 연구를 보면 45도 이상 구간은 수명 저하와 안전성 리스크를 함께 확인하는 편이므로, 고온 이벤트는 단순 일시 부하로만 보지 않는 편이 좋습니다."
- "구조 해석 결과의 안전계수는 내부 기준뿐 아니라 최근 양산 사례와 함께 비교하면 설명력이 높아집니다."

## 해석 시 주의할 점

- 논문 조건과 실제 차량 조건은 다를 수 있습니다.
- 실험실 데이터와 양산 환경은 차이가 큽니다.
- 특정 회사 발표 자료는 홍보 성격이 섞일 수 있습니다.
- 한 편의 논문만 보고 일반화하지 않습니다.
- 서로 다른 출처를 두 개 이상 교차 확인하면 더 안전합니다.

## 주의사항

- `/deep-research`는 VSCode extension 채팅창에서 쓰는 기능입니다.
- 강의장 네트워크가 막혀 있으면 실행이 안 될 수 있습니다.
- 외부 데이터는 반드시 **출처(URL + 날짜)** 를 함께 기록합니다.
- 본인 판단이 우선입니다. AI 요약은 초안이고 최종 결정은 수강생 몫입니다.

## R&D 부서 한정 팁

- R&D는 새 기술 동향에 민감하므로 3교시 실무 반복 루프에 `/deep-research`를 항상 옵션으로 둡니다.
- 매주 반복되는 외부 리서치 1건만 해도 내부 로그 해석의 품질이 올라갑니다.
- 한 번에 많은 주제를 보기보다 한 이슈를 깊게 보는 편이 더 실무적입니다.
