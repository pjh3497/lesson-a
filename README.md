# 성우하이텍 AI 마스터 과정 학습 레포

성우하이텍 AI 마스터 과정 MW4-5 수강생용 실습 저장소입니다.
이 레포는 비개발자 제조업 실무자가 Claude Code와 GitHub를 활용해 실제 업무 데이터를 분석하고,
그 결과를 정리하고, 저장하고, 공유하는 경험을 만들기 위해 구성되었습니다.

---

## 1. 이 레포의 목적

이 과정의 목표는 코딩을 잘하는 사람이 되는 것이 아닙니다.
목표는 제조업 현장의 실무자가 AI 도구를 이용해 자신의 업무를 더 빠르고 정확하게 수행하는 방법을 배우는 것입니다.

MW4에서는 설치와 연결, 기본 사용법, 아주 작은 실습부터 시작합니다.
MW5에서는 CLAUDE.md, 작업 규칙, 반복 가능한 워크플로우, 결과 정리, 간단한 배포, 그리고 마지막으로 나만의 스킬과 주간 점검 루틴 정리까지 이어집니다.

이 레포는 다음을 위해 설계되었습니다.

- Claude Code를 처음 접하는 수강생도 따라올 수 있도록 안내
- 부서별 실습 데이터를 바탕으로 실제 업무형 분석 수행
- 결과물을 GitHub에 저장하는 습관 형성
- 팀 단위 발표와 피드백을 위한 공통 구조 제공
- 강의 종료 후에도 계속 재사용 가능한 업무 템플릿 제공

---

## 2. 수강 대상

이번 과정의 수강생은 총 16명이며, 4개 그룹으로 나뉩니다.
모든 그룹은 제조업 기반 실무자이며, 개발자가 아닌 사용자를 전제로 설계되었습니다.

### 그룹 구성

- A그룹: R&D / CAE / 배터리 관련 실무
- B그룹: 전략 / 시장 / KPI / M&A 관련 실무
- C그룹: 제조 / 생산 / 품질 / 불량 분석 관련 실무
- D그룹: 경영 / 투자 / 구매 / 예산 관련 실무

### 전제 조건

- AI 도구를 업무에 적용해보고 싶다
- GitHub를 아직 익숙하게 쓰지 못해도 괜찮다
- 코딩을 몰라도 된다
- 중요한 것은 질문하는 능력과 결과를 정리하는 습관이다

## 3. Quick Start

아래 순서대로 진행하면 됩니다.

### Step 1. 레포 받기

```bash
git clone <이 저장소 주소>
cd lesson-a
```

### Step 2. VSCode에서 열기

```bash
code .
```

또는 VSCode에서 폴더 열기로 `lesson-a` 폴더를 직접 선택해도 됩니다.

### Step 3. Claude Code 실행 환경 준비

- Claude Team 계정 로그인
- Claude Code가 설치된 환경 확인
- VSCode 또는 터미널에서 Claude Code 실행

### Step 4. 첫 교시 시작

Claude Code에서 아래 명령으로 시작합니다.

```text
/start-mw4-1
```

이 명령은 MW4 1교시 실습의 시작점입니다.
설치 확인, 환경 체크, 첫 질문 방식까지 순서대로 안내받게 됩니다.

## 4. 코스 구조

과정은 MW4와 MW5, 총 8개 세션으로 구성됩니다.

### MW4 — 첫걸음: Claude Code 설치 + GitHub + 간단 실습

1. `mw4-1` 설치와 환경 점검
2. `mw4-2` Claude Code 기본 사용법 익히기
3. `mw4-3` 내 업무 데이터로 작은 분석 시도하기
4. `mw4-4` 결과 정리 후 GitHub에 저장하기

### MW5 — 심화: CLAUDE.md + 워크플로우 + Vercel 배포

1. `mw5-1` CLAUDE.md의 의미와 팀 작업 규칙 이해
2. `mw5-2` 부서별 반복 작업을 프롬프트로 구조화하기
3. `mw5-3` 결과물 폴더 정리와 발표 자료 연결하기
4. `mw5-4` 간단한 웹 결과물 또는 문서를 배포해보기

## 5. 학습 방식

이 과정은 “직접 해보는 방식”으로 진행됩니다.
Claude가 대신 다 해주는 수업이 아니라, 수강생이 직접 지시하고 확인하고 저장하는 수업입니다.

핵심 원칙은 다음과 같습니다.

- 작은 질문부터 시작한다
- 업무 맥락을 먼저 설명한다
- 결과를 바로 믿지 말고 검토한다
- 좋은 결과는 반드시 저장한다
- 저장한 결과는 다시 불러와 재사용한다

강의 중 자주 쓰게 될 표현은 아래와 같습니다.

- “이 파일 기준으로 분석해줘”
- “핵심만 5줄로 정리해줘”
- “이 내용을 표로 바꿔줘”
- “저장해줘”
- “올려줘”

## 6. 필수 준비물

수강 전 아래 준비물을 확인해 주세요.

### 필수

- Claude Team 계정
- VSCode 설치
- Git 설치

- 인터넷 연결

### 권장

- 회사 업무 예시 파일 또는 샘플 데이터
- 실습 결과를 메모할 노트

### 확인 포인트

- VSCode가 정상 실행되는가
- GitHub 로그인이 가능한가
- Claude Code를 실행할 수 있는가
- 실습 폴더에 접근 가능한가

## 7. 디렉토리 설명

이 레포는 실습용 결과물을 쉽게 정리할 수 있도록 폴더가 미리 준비되어 있습니다.

### 주요 폴더

- `company-context/`
  - 회사 시나리오, 부서 설명, 도구 사용 가이드가 들어 있습니다.
- `templates/`
  - 분석 결과, 발표 자료, CLAUDE.md 작성을 위한 템플릿이 들어 있습니다.
- `practice-data/`
  - 부서별 실습 데이터가 들어가는 폴더입니다.
  - 실제 수업에서는 각 그룹이 자신의 데이터를 이 폴더 기준으로 참조합니다.
- `organized/`
  - 정리된 결과물을 저장하는 폴더입니다.
- `analysis/`
  - 분석 중간 결과물을 저장하는 폴더입니다.
- `reviews/`
  - 검토, 피드백, 발표 리허설 메모를 저장하는 폴더입니다.

---

## 8. 수강생이 기억해야 할 작업 흐름

가장 기본적인 흐름은 아래와 같습니다.

1. 데이터를 확인한다
2. Claude에게 맥락을 설명한다
3. 원하는 분석 결과 형식을 말한다
4. 초안을 받는다
5. 수정 지시를 한다
6. 만족하면 저장한다
7. GitHub에 올린다

즉, 핵심은 다음 2문장입니다.

- 저장해줘
- 올려줘

수업에서는 이 두 문장을 실제 습관으로 만드는 것이 중요합니다.

---

## 9. 그룹별 기대 결과물

각 그룹은 강의 종료 전 최소 1개의 업무형 AI 결과물을 만들어야 합니다.

예시:

- A그룹: 배터리 또는 CAE 관련 비교 분석 보고서
- B그룹: 시장 동향 / KPI / 경쟁사 비교 정리
- C그룹: 불량 원인 분석 또는 생산 리포트 초안
- D그룹: 예산 검토안, 구매 비교안, 투자 판단 메모

결과물은 Markdown 문서, 요약 보고서, 발표 자료 초안, 간단한 웹 결과물 등 다양한 형태가 가능합니다.

## 10. 추천 학습 태도

이 과정을 잘 따라오는 사람의 특징은 세 가지입니다.

- 모르면 바로 묻는다
- 한 번에 크게 시키지 않고 단계적으로 시킨다
- 결과를 보고 다시 수정 요청한다

반대로 가장 흔한 실수는 다음과 같습니다.

- AI가 알아서 다 해주길 기대한다
- 맥락 설명 없이 결과만 요구한다
- 저장하지 않고 창을 닫아버린다

## 11. 수업 운영 메모

- 본 레포는 강의용 기본 구조입니다.
- 실제 실습 파일은 수업 중 순차적으로 추가될 수 있습니다.
- 강사 안내에 따라 그룹별 작업 폴더나 파일명이 달라질 수 있습니다.

준비가 되었다면 이제 Claude Code에서 아래 명령을 입력하세요.

```text
/start-mw4-1
```

첫 교시의 목표는 “설치가 끝난 상태”가 아니라,
“내가 Claude Code에게 업무형 질문을 시작할 수 있는 상태”를 만드는 것입니다.

작게 시작해도 괜찮습니다.
중요한 것은 직접 해보는 것입니다.
